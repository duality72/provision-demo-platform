# Provision Demo Platform

The target platform repository for [Provision Demo](https://github.com/your-org/provision-demo). This repo receives automated pull requests containing connector configurations, generated by the Provision onboarding workflow.

## How It Works

1. A user submits a connector onboarding request through the Provision web app
2. The app dispatches a `workflow_dispatch` event to this repository
3. The GitHub Actions workflow decrypts the payload, generates configuration files, and opens a PR
4. A reviewer approves the PR to complete onboarding
5. Terraform discovers the new connector via `fileset()` and registers it

## Directory Structure

```
provision-demo-platform/
├── .github/
│   ├── scripts/
│   │   └── generate_connector.py   # Generates connector config files
│   └── workflows/
│       └── onboard-connector.yml   # Workflow dispatched by the app
├── connectors/                     # Auto-generated connector configs
│   └── demo-postgres/
│       ├── config.json             # Connector configuration
│       └── secrets.enc.json        # SOPS-encrypted secrets
└── terraform/
    ├── providers.tf
    ├── main.tf                     # Discovers connectors via fileset()
    ├── variables.tf
    ├── outputs.tf
    └── modules/
        └── connector/              # Module instantiated per connector
            ├── main.tf
            ├── variables.tf
            └── outputs.tf
```

## Connector Types

| Type | Config Fields | Secret Fields |
|------|--------------|---------------|
| `s3` | `bucket_name`, `region` | (none) |
| `postgres` | `host`, `port`, `database` | `username`, `password` |
| `rest-api` | `base_url`, `polling_schedule` | `api_key` |
| `sftp` | `host`, `port` | `username`, `ssh_private_key` |

## Workflow Inputs

The `onboard-connector.yml` workflow accepts these inputs via `workflow_dispatch`:

| Input | Type | Description |
|-------|------|-------------|
| `connector_name` | string | Name of the connector (used as directory name) |
| `connector_type` | string | One of: s3, postgres, rest-api, sftp |
| `encrypted_payload` | string | Base64-encoded age-encrypted JSON payload |
| `requested_by` | string | Email of the requesting user |

## Required Secrets

These are set by the provision-demo Terraform on this repository:

| Secret | Description |
|--------|-------------|
| `AGE_SECRET_KEY` | Age private key for decrypting payloads |
| `SOPS_KMS_ARN` | KMS key ARN for SOPS encryption |
| `AWS_ROLE_ARN` | IAM role ARN for GitHub Actions OIDC |

## Terraform Usage

The Terraform configuration automatically discovers connectors:

```
cd terraform
terraform init
terraform plan
terraform apply
```

Each `connectors/*/config.json` file is discovered via `fileset()` and an instance of the `connector` module is created for each one.

## Adding a New Connector Type

To add a new connector type:

1. Add the type definition to `CONNECTOR_REGISTRY` in `.github/scripts/generate_connector.py`
2. Add the corresponding type to the provision-demo frontend registry (`CONNECTOR_REGISTRY` in `index.html`)
3. Add the type to the dispatch validation in `dispatch.py`

All three registries must match for the system to work correctly.
