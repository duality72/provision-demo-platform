# Provision Demo Platform

The target platform repository for [Provision Demo](https://github.com/duality72/provision-demo). This repo receives automated pull requests for connector onboarding and removal, generated by the Provision web application.

## How It Works

### Connector Onboarding

1. A user submits a connector onboarding request through the Provision web app
2. The app dispatches a `workflow_dispatch` event to the `onboard-connector.yml` workflow
3. The workflow decrypts the age-encrypted payload using the secret key
4. A Python script generates connector configuration files (config.json + SOPS-encrypted secrets)
5. The workflow creates a branch, commits the files, and opens a PR
6. A reviewer approves and merges the PR to complete onboarding
7. Terraform discovers the new connector via `fileset()` and registers it

### Connector Removal

1. A user clicks "Remove" on an active connector in the Provision web app
2. The app dispatches a `workflow_dispatch` event to the `remove-connector.yml` workflow
3. The workflow deletes the connector's directory and opens a PR
4. A reviewer approves and merges the PR to complete removal

### Branch Naming

All branches use timestamped names to avoid conflicts:
- Onboarding: `feat/onboard-{name}-YYYYMMDD-HHMMSS`
- Removal: `feat/remove-{name}-YYYYMMDD-HHMMSS`

This allows connectors to be added, removed, and re-added without branch name collisions.

## Directory Structure

```
provision-demo-platform/
├── .github/
│   ├── scripts/
│   │   └── generate_connector.py      # Generates connector config files
│   └── workflows/
│       ├── onboard-connector.yml      # Onboarding workflow (dispatched by app)
│       └── remove-connector.yml       # Removal workflow (dispatched by app)
├── connectors/                        # Auto-generated connector configs
│   └── {connector-name}/
│       ├── config.json                # Connector configuration
│       └── secrets.enc.json           # SOPS-encrypted secrets (if applicable)
└── terraform/
    ├── providers.tf
    ├── main.tf                        # Discovers connectors via fileset()
    ├── variables.tf
    ├── outputs.tf
    └── modules/
        └── connector/                 # Module instantiated per connector
            ├── main.tf
            ├── variables.tf
            └── outputs.tf
```

## Connector Types

| Type | Config Fields | Secret Fields |
|------|--------------|---------------|
| `s3` | `bucket_name`, `region` | (none) |
| `postgres` | `host`, `port`, `database` | `username`, `password` |
| `rest-api` | `base_url`, `polling_schedule` | `api_key` |
| `sftp` | `host`, `port` | `username`, `ssh_private_key` |

## Workflows

### onboard-connector.yml

Dispatched when a user submits a new connector through the web app.

**Inputs:**

| Input | Type | Description |
|-------|------|-------------|
| `connector_name` | string | Name of the connector (used as directory name) |
| `connector_type` | string | One of: s3, postgres, rest-api, sftp |
| `encrypted_payload` | string | Base64-encoded age-encrypted JSON payload |
| `requested_by` | string | Email of the requesting user |
| `branch_name` | string | Timestamped branch name generated by the Lambda |

**Steps:**
1. Checkout repository
2. Install age and SOPS
3. Configure AWS credentials (OIDC)
4. Decrypt the age-encrypted payload
5. Run `generate_connector.py` to create config files
6. Clean up sensitive files
7. Create branch, commit, and push
8. Open a PR

### remove-connector.yml

Dispatched when a user clicks "Remove" on an active connector in the web app.

**Inputs:**

| Input | Type | Description |
|-------|------|-------------|
| `connector_name` | string | Name of the connector to remove |
| `requested_by` | string | Email of the requesting user |
| `branch_name` | string | Timestamped branch name generated by the Lambda |

**Steps:**
1. Checkout repository
2. Verify connector directory exists
3. Delete the connector directory (`git rm -r`)
4. Create branch, commit, and push
5. Open a PR

## Required Secrets

These are set by the provision-demo Terraform on this repository:

| Secret | Description |
|--------|-------------|
| `AGE_SECRET_KEY` | Age private key for decrypting payloads |
| `SOPS_KMS_ARN` | KMS key ARN for SOPS encryption |
| `AWS_ROLE_ARN` | IAM role ARN for GitHub Actions OIDC |

## Terraform Usage

The Terraform configuration automatically discovers connectors:

```
cd terraform
terraform init
terraform plan
terraform apply
```

Each `connectors/*/config.json` file is discovered via `fileset()` and an instance of the `connector` module is created for each one. When a connector directory is removed (via a merged removal PR), the next `terraform apply` destroys the corresponding resources.

## Adding a New Connector Type

To add a new connector type, changes are needed in three places:

1. **provision-demo-platform**: Add the type definition to `CONNECTOR_REGISTRY` in `.github/scripts/generate_connector.py`
2. **provision-demo (frontend)**: Add the type to `CONNECTOR_REGISTRY` in `index.html`, and add entries to `FIELD_VALIDATORS` and `SECRET_TOOLTIPS`
3. **provision-demo (backend)**: Add the type to `CONNECTOR_TYPES` in `dispatch.py`

All three registries must match for the system to work correctly.
